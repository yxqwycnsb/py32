# STM32 简单磨刀器 — 项目文档

## 1. 项目概述

- **主控芯片**：STM32F103C8T6
- **操作系统**：FreeRTOS
- **电机驱动**：两路 A4988 步进电机驱动（电机1 + 电机2）
- **位置反馈**：AS5600 磁编码器（I2C 通信）
- **人机交互**：1个按键 + 1个LED + USART1串口调试输出
- **功能目标**：用户手动推动磨刀行程一次 → 系统自动测量行程 → 电机2自动往返打磨5次 → 电机1额外进给3圈 → 全部复位归零

---

## 2. 硬件引脚定义

### 2.1 电机1 — A4988（定时器脉冲）

| 信号 | 引脚 | 模式 | 说明 |
|------|------|------|------|
| **STEP1** | PA1 | AF推挽输出 | 脉冲信号，由 TIM2 通道生成 |
| **DIR1** | PA6 | 推挽输出 | 方向控制：高=正转，低=反转 |
| **EN1** | PA7 | 推挽输出 | 使能控制：低电平有效，使能电机 |
| **MS1** | PB5 | 推挽输出 | 细分设置（1/16 细分模式） |

### 2.2 电机2 — A4988（软件脉冲）

| 信号 | 引脚 | 模式 | 说明 |
|------|------|------|------|
| **STEP2** | PA0 | 推挽输出 | 脉冲信号，由软件 GPIO 翻转生成 |
| **DIR2** | PA4 | 推挽输出 | 方向控制：高=正转，低=反转 |
| **EN2** | PA5 | 推挽输出 | 使能控制：低电平有效，使能电机 |

> **注意**：电机1和电机2共用 MS1（PB5）细分配置引脚，均工作于 1/16 细分模式。
> 电机步进角 1.8°（200步/圈），1/16 细分后 = **3200 脉冲/圈**。

### 2.3 AS5600 磁编码器

| 信号 | 引脚 | 模式 | 说明 |
|------|------|------|------|
| **SCL** | PB6 | 开漏输出 | I2C1 时钟线 |
| **SDA** | PB7 | 开漏输出 | I2C1 数据线 |
| I2C地址 | 0x36 << 1 | — | 7位地址左移1位 |
| 分辨率 | 12位（0~4095） | — | 对应0°~360° |

### 2.4 按键 & LED

| 信号 | 引脚 | 模式 | 说明 |
|------|------|------|------|
| **按键** | PB1 | 上拉输入 | 按下为低电平（Bit_RESET） |
| **LED** | PC13 | 推挽输出 | **低电平点亮**（常见开发板取反逻辑） |

### 2.5 串口调试

| 信号 | 引脚 | 模式 | 说明 |
|------|------|------|------|
| **TX** | PA9 | 复用推挽输出 | USART1 发送 |
| **RX** | PA10 | 浮空输入 | USART1 接收 |
| 波特率 | 115200 | — | 8N1 无校验 |

## 3. 功能说明

### 3.1 工作流程概览

```
用户设定零点 → 手动推动刀具走一个行程 → 
系统自动测量行程距离 → 电机2自动往返慢速打磨5次 → 
电机1额外前进3圈后返回 → 电机2恢复到目标位置 → 流程结束
```

### 3.2 状态机总览

```
ST_IDLE ──①──▶ ST_WAIT_ZERO ──②──▶ ST_MONITOR_MOVE ──③──▶ ST_RETURN_HOME
                                                                      │
                     ┌──────────────────────────────────────────────────┘
                     ▼
            ST_M2_REPEAT_OUT ◀─────────────────────┐
                     │                              │  × 5 次
                     ▼                              │
            ST_M2_REPEAT_BACK ──────────────────────┘
                     │（5次完成）
                     ▼
            ST_M1_FINAL_OUT ──▶ ST_M1_FINAL_BACK
                                          │
                                          ▼
                                 ST_M2_RESET_OUT ──▶ ST_M2_RESET_BACK
                                                               │
                                                               ▼
                                                        ST_FINISHED
                                                               │
                                                               ▼
                                                          ST_IDLE
```

## 4. 程序逻辑 — 各状态详解

### 4.1 ST_IDLE — 空闲状态

| 项目 | 内容 |
|------|------|
| **功能** | 上电默认状态，等待用户按下按键设定零点 |
| **LED状态** | 熄灭（`led_set(false)`） |
| **触发条件** | 上电默认进入 |
| **跳转条件** | 检测到按键**下降沿**且编码器读数有效 → 进入 `ST_WAIT_ZERO` |
| **串口输出** | 初始无输出；跳转时输出 `[STATE] ZERO_SET` |

### 4.2 ST_WAIT_ZERO — 等待零点确认

| 项目 | 内容 |
|------|------|
| **功能** | 零点已设置，LED闪烁提示用户，等待再次按键开始监测 |
| **LED状态** | 500ms 周期闪烁（`led_blink(now, 500)`） |
| **触发条件** | 从 ST_IDLE 跳入 |
| **跳转条件** | 按键下降沿 → 重新取样起始位置 → 进入 `ST_MONITOR_MOVE` |
| **串口输出** | 跳转时输出 `[STATE] START_MONITOR` |

### 4.3 ST_MONITOR_MOVE — 监测推动行程

| 项目 | 内容 |
|------|------|
| **功能** | 用户手动推动刀具，系统通过编码器实时监测角度变化 |
| **LED状态** | 100ms 周期快速闪烁 |
| **监测逻辑** | 持续跟踪 `continuous_raw`（连续角度，已处理360°回绕），与起始值比较 |
| **完成条件** | 编码器角度 **5秒内无变化**（`encoder_stable_ms = 5000`） |
| **超时条件** | 30秒内仍未完成监测（`encoder_timeout_ms = 30000`） |
| **跳转条件** | 稳定超时 → `ST_RETURN_HOME`；总超时 → `ST_ERROR` |
| **串口输出** | 正常：`[STATE] END_CAPTURE`；异常：`[STATE] ERROR_TIMEOUT` |

### 4.4 ST_RETURN_HOME — 回归起点

| 项目 | 内容 |
|------|------|
| **功能** | 电机2反向转动，将刀具退回到起始位置 |
| **LED状态** | 常亮 |
| **运动控制** | 电机2以 20 RPM 反转，直到 `continuous_raw == start_continuous_raw` |
| **超时条件** | 30秒（`move_timeout_ms = 30000`） |
| **跳转条件** | 到达起点 → `ST_M2_REPEAT_OUT`；超时 → `ST_ERROR` |
| **串口输出** | 正常：`[STATE] HOME_DONE` |

### 4.5 ST_M2_REPEAT_OUT / ST_M2_REPEAT_BACK — 电机2往返打磨

| 项目 | 内容 |
|------|------|
| **功能** | 电机2正转走到目标步数 → 反转回到起点，计为1次往返，**重复5次** |
| **LED状态** | 常亮 |
| **OUT** | 电机2正转，目标步数 = 测量步数 `measured_steps` |
| **BACK** | 电机2反转，直到 `continuous_raw == start_continuous_raw` |
| **循环次数** | 共 5 次（`repeat_times = 5`） |
| **超时条件** | 每次 30 秒 |
| **跳转条件** | 5次完成 → `ST_M1_FINAL_OUT` |
| **串口输出** | OUT完成：`[STATE] M2_OUT_DONE`；BACK完成：`[STATE] M2_LOOP_NEXT` 或 `[STATE] M2_REPEAT_DONE` |

### 4.6 ST_M1_FINAL_OUT / ST_M1_FINAL_BACK — 电机1最终前进

| 项目 | 内容 |
|------|------|
| **功能** | 电机1正转前进 3 圈（`motor1_turn_steps = 3 × steps_per_rev`），再反转返回起点 |
| **LED状态** | 常亮 |
| **跳转条件** | 到达起点 → `ST_M2_RESET_OUT` |
| **串口输出** | OUT完成：`[STATE] M1_FINAL_OUT_DONE`；BACK完成：`[STATE] M1_FINAL_BACK_DONE` |

### 4.7 ST_M2_RESET_OUT / ST_M2_RESET_BACK — 电机2最终复位

| 项目 | 内容 |
|------|------|
| **功能** | 电机2正转走回目标步数位置，再反转回到起点，恢复初始对准 |
| **LED状态** | 常亮 |
| **跳转条件** | BACK到达起点 → `ST_FINISHED` |
| **串口输出** | OUT完成：`[STATE] M2_RESET_OUT_DONE`；BACK完成：`[STATE] M2_RESET_DONE` |

### 4.8 ST_FINISHED — 完成

| 项目 | 内容 |
|------|------|
| **功能** | 所有动作完成，停止所有电机，回到空闲状态 |
| **LED状态** | 熄灭 |
| **串口输出** | `[STATE] FINISHED` |

### 4.9 ST_ERROR — 错误状态

| 项目 | 内容 |
|------|------|
| **功能** | 超时等异常情况，停止所有电机 |
| **LED状态** | 熄灭 |
| **恢复方式** | 任务循环中不做恢复，需复位单片机 |

## 5. 关键业务参数

| 参数名 | 值 | 说明 |
|--------|-----|------|
| `motor1_rpm` | **20 RPM** | 电机1运行转速 |
| `motor2_rpm` | **20 RPM** | 电机2运行转速 |
| `motor1_turn_steps` | **3 圈 = 9600步** | 电机1最终前进总步数（3200步/圈） |
| `repeat_times` | **5 次** | 电机2往返打磨次数 |
| `encoder_timeout_ms` | **30000 ms** | 监测行程阶段总超时 |
| `move_timeout_ms` | **30000 ms** | 各运动阶段超时 |
| `encoder_stable_ms` | **5000 ms** | 角度稳定判定时间（秒内无变化） |
| `steps_per_rev` | **3200** | 每圈步数（200×16细分） |

## 6. 串口调试指南

### 6.1 连接方式

- 使用 USB-TTL 串口模块连接 PA9 (TX)、PA10 (RX)、GND
- **波特率**：**115200**，数据位8，停止位1，无校验
- 建议使用串口助手软件（如 SSCOM、PuTTY、MobaXterm）
- **重要**：USB-TTL 模块的 **TX 接 PA10 (RX)**，模块的 **RX 接 PA9 (TX)**，**GND 必须共地**

### 6.2 启动日志（上电后立即输出）

代码在 `usart_init` 之后会立即输出启动日志，用于确认系统是否正常运行：

**正常启动时的串口输出**：
```
[BOOT] AS5600 init OK
[BOOT] Waiting for first encoder reading...
[BOOT] First encoder reading OK, entering main loop
STATE=IDLE BTN_RAW=1 BTN_PRESS=0 ...
```

**如果 AS5600 初始化失败**：
```
[BOOT] ERROR: AS5600 init failed! Check I2C wiring (PB6/PB7)
（然后 LED 会快速闪烁3次，停顿1秒，循环往复）
```

### 6.3 卡在 "Waiting for first encoder reading..." — 排查方法

如果你看到以下输出后程序就停住了：

```
[BOOT] AS5600 init OK
[BOOT] Waiting for first encoder reading...
```

说明 **AS5600 驱动初始化成功（STM32 的 I2C 外设已配置），但是读不到芯片数据**。代码会每秒输出一条诊断日志：

```
[BOOT] Still waiting for encoder... retry=100 | Check: PB6/PB7 4.7k pullup to 3.3V? AS5600 powered? Magnet present?
```

🔴 **最常见原因：PB6 和 PB7 缺少 4.7kΩ 上拉电阻接 3.3V**

I2C 协议使用开漏输出，SCL 和 SDA 线**必须有外部上拉电阻**才能工作。STM32 的内部上拉电阻（约 40kΩ）太弱，无法满足 I2C 时序要求。按以下顺序排查：

| # | 检查项 | 排查方法 |
|---|--------|----------|
| **1** | **PB6/PB7 上拉电阻** | 用万用表电阻档分别测 PB6 ↔ 3.3V 和 PB7 ↔ 3.3V，应为 **4.7kΩ** 左右。如果断路或阻值很大，说明缺上拉电阻 |
| **2** | **AS5600 模块供电** | 用万用表测模块 VCC-GND 电压，应为 3.3V（或 5V，取决于模块版本） |
| **3** | **SCL ↔ SDA 接线正确** | PB6 = SCL，PB7 = SDA。确认杜邦线没有接反或松动 |
| **4** | **I2C 地址** | AS5600 默认 7 位地址 0x36。检查模块背面 AD0/AD1 焊盘是否被意外改动导致地址变化 |
| **5** | **磁铁位置** | AS5600 芯片正上方 0.5~3mm 处需要一颗径向磁铁。注意：即使没有磁铁，芯片也应该能通过 I2C 返回数据（只是角度值无意义） |

#### 上拉电阻接线方法

```
         3.3V
          │
     ┌────┴────┐
     │         │
  ┌──┴──┐   ┌──┴──┐
  │4.7kΩ│   │4.7kΩ│
  └──┬──┘   └──┬──┘
     ├─ PB6    ├─ PB7
  (SCL)    (SDA)
     │         │
     └────┬────┘
         GND
```

> **快速修复**：在 PB6 和 3.3V 之间焊一个 4.7kΩ 电阻，PB7 和 3.3V 之间同样焊一个 4.7kΩ 电阻。这是 I2C 标准要求，**加了之后 95% 的情况都能正常通信**。

### 6.4 串口完全无输出 — 排查方法

如果上电后串口助手**完全无任何输出**（连 `[BOOT] AS5600 init OK` 都没有），按以下顺序逐项排查：

| # | 检查项 | 方法 | 常见错误 |
|---|--------|------|----------|
| 1 | **程序是否烧录成功** | 检查 Keil 编译无错误，ST-Link/串口烧录完成后按复位键 | 只编译了但忘记点击下载按钮；芯片被锁住 |
| 2 | **芯片是否供电** | 用万用表测 3.3V 和 GND 之间电压 | 电源接反；USB线仅供电无数据；LDO 损坏 |
| 3 | **晶振是否起振** | 用示波器测 OSC_IN/OSC_OUT 波形，或用万用表测 PA9 电压是否为高电平（空闲态应为 3.3V） | 晶振虚焊；复位电容焊接不良导致一直处于复位状态 |
| 4 | **复位引脚电平** | NRST 引脚应为高电平（3.3V） | 复位按键短路；外部电路拉低 |
| 5 | **BOOT0/BOOT1 模式** | BOOT0 = 0（GND），BOOT1 = 0（GND）→ 从 Flash 启动 | BOOT0 误接 3.3V（进入 ISP 模式不运行用户程序） |
| 6 | **TX/RX 接线** | USB-TTL 的 TX → PA10 (RX)，USB-TTL 的 RX → PA9 (TX) | **最常见的错误**：TX 接 TX、RX 接 RX，或只接了 TX ↔ PA9（正确应为 RX ↔ PA9） |
| 7 | **GND 共地** | USB-TTL 的 GND 必须与 STM32 的 GND 相连 | 忘了接 GND，串口电平无参考 |
| 8 | **波特率** | 串口助手设为 **115200** / 8N1 | 设为 9600 或其他波特率，读到乱码或无输出 |
| 9 | **TX 引脚复用冲突** | PA9 是否有其他外设占用 | 检查硬件是否有其他芯片共用 PA9 |
| 10 | **FreeRTOS 堆栈不足** | 任务栈太小导致任务创建失败 | 尝试将 `xTaskCreate` 的栈大小从 256 增加到 512 |

#### 快速验证：串口回环测试

如果要确认是 STM32 的串口问题还是 USB-TTL 模块的问题：

1. 把 USB-TTL 模块的 **TX 和 RX 短接**
2. 在串口助手中发送任意字符
3. 如果收到相同字符返回 → USB-TTL 模块正常，问题出在 STM32 端
4. 如果无返回 → USB-TTL 模块有故障

#### 启动过程串口信号详解

当程序正常运行时，`usart_init` 执行后会完成以下操作：
- RCC 打开 GPIOA 和 USART1 时钟
- PA9 配置为**复用推挽输出**（AF_PP），这是 USART1_TX 的功能
- PA10 配置为**浮空输入**（IN_FLOATING），这是 USART1_RX 的功能
- USART1 波特率设为 115200，8N1，无流控
- USART1 使能

如果这些步骤都成功，`usart_send_string("[BOOT] AS5600 init OK")` 就能正常输出。

### 6.5 周期性调试信息（每 200ms 输出一次）

系统每 200ms 自动输出一行调试数据，格式如下：

```
STATE=MONITOR BTN_RAW=1 BTN_PRESS=0 BTN_FALL=0 ENC_OK=1 RAW=1024 CONT=1024 START=512 END=512 STEP=0 M2_STEP=0 REP=0 M1=0 M2=0
```

| 字段 | 含义 | 说明 |
|------|------|------|
| `STATE` | 当前状态机状态 | 如 IDLE、MONITOR、M2_OUT 等 |
| `BTN_RAW` | 按键原始电平 | 1=未按下，0=按下（PB1上拉） |
| `BTN_PRESS` | 按键当前状态 | 1=按下，0=未按下 |
| `BTN_FALL` | 按键下降沿 | 1=检测到下降沿（松开瞬间） |
| `ENC_OK` | 编码器读数有效 | 1=有效，0=读取失败 |
| `RAW` | 编码器原始值 | 0~4095，对应0°~360° |
| `CONT` | 连续角度 | 已处理360°回绕，可跨圈累加 |
| `START` | 起始原始角度 | 零点设定时的编码器值 |
| `END` | 终点原始角度 | 监测完成时的编码器值 |
| `STEP` | 测量步数 | 从起点到终点的脉冲步数 |
| `M2_STEP` | 目标步数 | 电机2需要走的步数 |
| `REP` | 往返计数 | 当前已完成往返次数 |
| `M1` | 电机1使能 | 1=运行中，0=停止 |
| `M2` | 电机2使能 | 1=运行中，0=停止 |

### 6.6 各步骤串口输出与故障排查

#### 步骤 1 — 上电初始化

**预期输出**：
```
STATE=IDLE BTN_RAW=1 BTN_PRESS=0 ...
```

**故障排查**：
- 如果无任何输出 → 检查串口连接（PA9 TX）、波特率 115200、GND
- 如果 `ENC_OK=0` → AS5600 初始化失败，检查 I2C 接线（PB6/PB7），检查 3.3V/GND 供电
- 如果一直停在 IDLE 且 `BTN_RAW` 始终为 0 → PB1 短路或外部拉低，检查按键电路

---

#### 步骤 2 — 按下按键设定零点

**操作**：按下按键后松开

**预期输出**：
```
[STATE] ZERO_SET
STATE=WAIT_ZERO ...
```

**此时 LED**：500ms 周期闪烁

**故障排查**：
- 按键无响应 → 检查 `BTN_FALL` 是否从 0→1（松开上升沿），确认 PB1 上拉是否正常
- `BTN_RAW` 始终为 0 → PB1 被拉低，检查按键是否卡住或电路短路

---

#### 步骤 3 — 再次按键开始监测

**操作**：再次按下按键后松开

**预期输出**：
```
[STATE] START_MONITOR
STATE=MONITOR ...
```

**此时 LED**：100ms 快速闪烁

**故障排查**：
- 无变化 → 同上，检查按键

---

#### 步骤 4 — 推动刀具

**操作**：手动推动刀具走完一个研磨行程，然后保持静止

**预期行为**：
- 串口中 `CONT` 值会持续变化
- 停止推动后，`STEP` 值会稳定
- 约 **5秒** 无变化后自动进入下一步

**预期输出**：
```
[STATE] END_CAPTURE
STATE=RETURN_HOME ...
```

**故障排查**：

| 现象 | 原因 | 解决方法 |
|------|------|----------|
| 推动时 `CONT` 不变 | AS5600 未检测到转动 | 检查磁铁是否对准编码器芯片中心，距离 <3mm |
| `CONT` 跳变剧烈 | I2C 干扰或接触不良 | 检查 PB6/PB7 接线，加 4.7kΩ 上拉电阻 |
| `ENC_OK=0` 频繁出现 | I2C 通信错误 | 降低 I2C 速率，检查线长，尝试加屏蔽 |
| 30秒后输出 `ERROR_TIMEOUT` | 推动未在30秒内完成 | 重新操作，确保在30秒内完成推动并静止 |
| 5秒后仍不跳转 | 有微小振动导致角度持续变化 | 确保刀具完全静止，或减小 `encoder_stable_ms` |

---

#### 步骤 5 — 电机2回归起点

**预期输出**：
```
[STATE] HOME_DONE
STATE=M2_OUT ...
```

**串口字段变化**：`M2=1` 表示电机2正在运行

**故障排查**：

| 现象 | 原因 | 解决方法 |
|------|------|----------|
| 电机2不转 | EN2 未使能或脉冲无输出 | 检查 PA0/PA4/PA5 接线 |
| 30秒超时 `ERROR_TIMEOUT` | 电机无法到达起点 | 检查 `CONT` 是否朝着 `START` 值变化 |
| 电机堵转 | 电流不足或机械卡死 | 调高 A4988 驱动电流，检查机械 |
| 电机越过起点不停 | 编码器读数异常 | 检查 AS5600，确认 `CONT` 值能准确回到 `START` |

---

#### 步骤 6 — 电机2往返打磨（5次）

**预期输出**（每完成一次往返）：
```
[STATE] M2_OUT_DONE
[STATE] M2_LOOP_NEXT        ← 第1-4次后
[STATE] M2_OUT_DONE
[STATE] M2_REPEAT_DONE      ← 第5次后
STATE=M1_FOUT ...
```

**故障排查**：

| 现象 | 原因 | 解决方法 |
|------|------|----------|
| 往返次数不对 | 脉冲计数不准 | 检查 `M2_STEP` 值是否合理 |
| 电机2方向反了 | DIR2 电平逻辑反了 | 交换电机2的 A+/A- 或 B+/B- 接线 |
| 30秒超时 | 步数太大，转速太慢 | 确认 `measured_steps` 合理 |

---

#### 步骤 7 — 电机1前进3圈

**预期输出**：
```
[STATE] M1_FINAL_OUT_DONE
[STATE] M1_FINAL_BACK_DONE
STATE=M2_RST_OUT ...
```

**串口字段变化**：`M1=1` 表示电机1正在运行

**故障排查**：

| 现象 | 原因 | 解决方法 |
|------|------|----------|
| 电机1不转 | 定时器 TIM2 未输出脉冲 | 检查 PA1 STEP 引脚，确认 TIM2 初始化 |
| 电机1方向反了 | DIR1 接线 | 检查 PA6 电平 |

## 8. 软件架构

```
main()
  └─ xTaskCreate(PhotoTask, "photo", stack=256, priority=2)
       └─ motor_pair_init()          // 初始化两路A4988
       └─ button_init()              // 初始化PB1按键
       └─ led_init()                 // 初始化PC13 LED
       └─ usart_init(115200)         // 初始化USART1
       └─ as5600_init()              // 初始化AS5600编码器
       └─ 主循环 (10ms周期)
            ├─ as5600_update()       // 读取编码器
            ├─ _angle_diff_raw()     // 处理角度回绕
            ├─ 状态机更新            // 10个状态的跳转逻辑
            ├─ 按键检测              // 下降沿检测
            ├─ LED状态更新           // 闪烁/常亮/熄灭
            └─ 串口调试输出          // 200ms周期
```

## 9. 注意事项

1. **AS5600 初始化失败** → LED 快闪3次+停顿报警，串口输出错误日志，不再静默死循环
2. **卡在 "Waiting for first encoder reading"** → **99% 是 PB6/PB7 缺 4.7kΩ 上拉电阻**，I2C 开漏线必须有外部上拉
3. **电机2默认上电不使能**（`a4988_motor2_enable(false)`），由状态机按需开启
4. **连续角度处理**：`_angle_diff_raw()` 处理 0⇄4095 的回绕，确保跨圈角度累加正确
5. **A4988 细分**：两路电机共用 MS1 引脚（PB5），当前配置为 1/16 细分
6. **按键去抖**：未在代码中实现硬件/软件去抖，依赖 10ms 任务周期自然过滤
7. **`ST_ERROR` 不可恢复**：进入错误状态后电机全停，需手动复位单片机
