================================================================================
                      PY32F002B简单全自动磨刀器正反转功能说明（当前版本）
================================================================================

一、商用控制功能（V1.0）
1) 电机驱动方案
   - 采用 RZ7889 电机驱动芯片，实现电机正转 / 反转控制；
   - 通过 MCU 控制方向与使能信号，完成启停与方向切换。

2) 调速能力
   - 通过 PWM 占空比调节实现电机转速控制；
   - 支持多档速度或连续调速（按项目参数配置）；
   - 具备软启动策略，降低启动冲击电流与机械冲击；
   - 机械约束：PWM 占空比不得低于 30%，低于 30% 时减速齿轮电机无法稳定运动。

3) 基础运行逻辑
   - 上电后执行运行指示（绿灯固定节奏翻转 3 次，每次间隔 2 秒）；
   - 采集开关输入、充电状态与 ADC 数据；
   - 基于状态机统一控制电机输出与指示灯显示。

4) 保护与可维护性
   - 保留过流/堵转保护与低电保护机制；
   - 增加刀点丢失超时保护（超时停机并进入重启门控）；
   - 支持串口调试输出（可选），便于产测与售后问题定位。

5) 刀点检测与自动换向
   - 通过凹槽光电对（光电开关）检测刀点基准位置；
   - 启动后先按设定方向运行，检测到刀点信号后进行圈数/角度计数；
   - 到达“不到一圈”的目标行程后自动反转，实现往复磨削控制；
   - 反转后同样以刀点信号作为位置基准，确保换向位置一致性。

6) 刀点控制参数（商用规格）
   - 目标行程：N 圈，且 N < 1.0（默认 N = 0.85，可调范围 0.70 ~ 0.95）；
   - 刀点有效沿：可配置为上升沿或下降沿（默认下降沿）；
   - 刀点去抖时间：5~20ms（默认 10ms）；
   - 丢刀点超时保护：在设定时间内未检测到刀点则停机告警（默认 1500ms）；
   - 反转时序：减速 -> 停机保持 -> 反向软启动（默认 80ms -> 120ms -> 200ms）；
   - 方向切换保护：未停机禁止直接从 F1 切换到 B1（或反向），避免驱动冲击；
   - PWM 最小占空比：30%（低于该值判定为无效转速命令，按 30% 下限执行）。

================================================================================
二、硬件资源与引脚定义（商用版本）
1) RZ7889 电机驱动控制引脚（MCU -> Driver）
   - PA0 : MOTOR_EN（驱动使能，0=关闭，1=使能）
   - PA1 : MOTOR_PWM（调速 PWM 输出，建议 16~24kHz，默认 20kHz）
   - PA4 : MOTOR_F1（方向控制 F1）
   - PA5 : MOTOR_B1（方向控制 B1）

   方向逻辑定义（按本项目硬件约定）：
   - 正转：F1=1，B1=0
   - 反转：F1=0，B1=1
   - 停机：F1=0，B1=0
   - 保护约束：禁止 F1 与 B1 同时为 1

2) 刀点检测与人机输入（Sensor/Input -> MCU）
   - PB4 : KNIFE_HOME_IN（凹槽光电对输入，刀点基准检测）
   - PB7 : SW_IN（开关输入，低电平表示 ON）
   - PB1 : TP4056_STDBY（充电待机状态，低电平有效）
   - PB2 : TP4056_CHRG（充电进行状态，低电平有效）

3) 指示输出（MCU -> LED）
   - PB3 : MOTOR_LED_G（绿灯）
   - PB0 : CHARGE_LED_R（红灯）

4) ADC 通道
   - ADC_CHANNEL_1（PA3）: 电流采样 I_SENSE
   - ADC_CHANNEL_3（PA6）: 电池电压采样 VBAT_SENSE

5) 量产接线与容差说明
   - 如实际 PCB 已固定其他 IO，请以原理图网络名为准，仅保持“功能定义”一致；
   - KNIFE_HOME_IN 建议开启输入滤波或软件去抖（默认 10ms），避免电磁干扰误触发；
   - MOTOR_PWM 频率建议避开可闻频段啸叫点，优先 20kHz 附近。

================================================================================
三、核心状态机（本次更新重点）
1) 按钮状态
   - APP_BUTTON_DISCONNECT：开关断开
   - APP_BUTTON_CONNECT：开关接通

2) 系统主状态（3个核心状态）
   - APP_SYS_NORMAL：正常使用
   - APP_SYS_CHARGING：充电中
   - APP_SYS_LOW_BATTERY：低电量保护

3) 电机状态
   - APP_MOTOR_STOP：停机
   - APP_MOTOR_RUN：运行
   - APP_MOTOR_OVERCURRENT：过流/堵转故障

4) LED 状态
   - 绿灯：灭 / 亮 / 闪烁
   - 红灯：灭 / 常亮 / 呼吸 / 低电闪烁

5) 新增“停机原因”与“重启门控”机制
   - APP_STOP_REASON_NONE
   - APP_STOP_REASON_LOW_BATTERY
   - APP_STOP_REASON_STALL
   - APP_STOP_REASON_KNIFE_TIMEOUT

   配套标志：
   - restartRequireButtonRelease：异常停机后，要求先松开按钮
   - startRequestLatched：启动请求锁存（由按钮“断开->接通”边沿触发）

================================================================================
四、关键行为逻辑（与旧版相比的主要变化）
1) 按钮与电机解耦
   - 按钮不再直接等于“电机立即运行”；
   - 按钮只用于产生“启动请求”（断开->接通沿）。

2) 异常停机后禁止自动重启
   - 若因低电或堵转停机，会进入重启门控；
   - 即使按钮一直保持接通，电机也不会自动恢复；
   - 必须先检测到按钮断开，再次接通后才允许重启。

3) 堵转（过流）保护
   - 阈值：5000mA；
   - 连续命中 3 次后判定堵转；
   - 堵转后停机并进入“需先断开按钮”的重启门控流程。

4) 低电保护（带进入确认+释放滞回）
   - 进入条件：VBAT 连续低于 3300mV 达 2 秒；
   - 释放条件：VBAT 高于 3450mV；
   - 进入低电后停机并进入“需先断开按钮”的重启门控流程。

5) 系统状态判定优先级
   - 低电优先 -> APP_SYS_LOW_BATTERY
   - 否则充电中 -> APP_SYS_CHARGING
   - 否则 -> APP_SYS_NORMAL

6) 调试模式兼容逻辑（重要）
   - 当 APP_UART_DEBUG_ENABLE = 1 时，PB4 被调试串口复用，刀点往复功能自动关闭；
   - 此时电机输出降级为“固定正转直驱”（RUN 时正转目标占空比，STOP 时全关），用于调试和产测。

================================================================================
五、输出控制规则
1) 电机使能 PA0
   - 仅在“存在有效启动请求 + 非低电锁定 + 非堵转故障”时启动；
   - 其余情况关闭。

2) 绿灯 PB3
   - 低电锁定：强制熄灭；
   - 堵转故障：按 250ms 周期闪烁；
   - 正常状态：按钮接通常亮，断开熄灭。

3) 红灯 PB0
   - 低电锁定：持续闪烁提示；
   - 充电中：呼吸效果；
   - 满电（未充电且 VBAT >= 4.12V）：常亮；
   - 其他：熄灭。

================================================================================
六、采样与换算
1) ADC 转毫伏
   - adc_mV = adc * 1500 / 4095

2) 电流换算（5mΩ 分流电阻）
   - current_mA = adc * (1500*1000) / (4095*5)

3) 电池电压反推（22k:10k 分压）
   - vbat_mV = adc_mV * (22+10) / 10

4) 电流处理
   - 开机零点校准；
   - 异常 ADC 值过滤；
   - 一阶低通滤波（1/8）。

================================================================================
七、可配置宏（当前默认）
1) APP_UART_DEBUG_ENABLE = 0
   - 0: 关闭串口调试
   - 1: 开启串口调试（PB4 与串口复用，刀点往复自动降级为直驱）

2) APP_ADC_ENABLE = 1
   - 0: 关闭 ADC 相关逻辑
   - 1: 开启 ADC 采样与保护

================================================================================
八、工程用途总结
本工程是一个带状态机与保护策略的开机测试控制程序，具备：
- 上电自检指示
- 按钮触发启动（边沿触发）
- 电流采样与堵转保护
- 电池采样与低电保护
- 充电状态指示（呼吸/满电常亮）
- 异常停机后“必须断开再接通”的防误重启机制
- 可选串口调试输出
