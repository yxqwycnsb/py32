# STM32RTOS 编码器简单磨刀器 — 项目文档

## 1. 项目概述

| 项目 | 内容 |
|------|------|
| 项目名称 | STM32RTOS 编码器简单磨刀器 |
| MCU 型号 | STM32F103C8 |
| 主频 | 72MHz |
| 操作系统 | FreeRTOS |
| 开发环境 | Keil MDK-ARM |
| 核心功能 | 按键触发的双电机动作流程，电机2支持闭环串口控制 |
| 电机1 | A4988 步进驱动，保留原有脉冲控制 |
| 电机2 | 新购闭环电机，通过 USART2 控制 |

## 2. 硬件连接

### 2.1 引脚分配

| 引脚 | 功能 | 连接目标 | 说明 |
|------|------|----------|------|
| PA1 | 电机1 STEP | A4988 STEP | TIM2 PWM 输出 |
| PA6 | 电机1 DIR | A4988 DIR | 方向控制 |
| PA7 | 电机1 EN | A4988 EN | 低电平有效 |
| PA2 | USART2_TX | 电机2 RX | 闭环电机通讯发送 |
| PA3 | USART2_RX | 电机2 TX | 闭环电机通讯接收 |
| PB1 | 按键 | 用户输入 | 低电平按下 |
| PB7 | 传感器输入 | 光电传感器 | 若当前工程仍保留传感器则用于检测 |
| PB5 | MS1 | A4988 微步设置 | 共用 |

### 2.2 串口分工

| 串口 | 用途 | 是否可同时使用 |
|------|------|----------------|
| USART1 | 电脑串口调试 | 可以 |
| USART2 | 电机2闭环通讯 | 可以 |

两者不冲突，前提是：
- USART1 接电脑调试串口
- USART2 只接闭环电机
- 不要把同一组引脚同时接到两个设备

## 3. 当前程序流程

### 3.1 总流程

当前主流程已经改成下面的逻辑：

1. 上电后电机2释放，不锁死
2. 按键触发流程开始
3. 电机2当前位置读取并清零
4. 用户手动推动电机2到任意角度
5. 程序监测电机2角度变化是否稳定
6. 稳定后，电机1顺时针转一圈
7. 电机1逆时针转一圈
8. 电机2回到 0 点
9. 重复 5 次
10. 超时或异常进入错误状态

### 3.2 状态机

| 状态 | 含义 |
|------|------|
| `ST_IDLE` | 空闲，等待按键 |
| `ST_WAIT_M2_STABLE` | 等待用户手动推位并稳定 |
| `ST_M1_CW` | 电机1 顺时针转一圈 |
| `ST_M1_CCW` | 电机1 逆时针转一圈 |
| `ST_M2_RETURN_ZERO` | 电机2 回零 |
| `ST_REPEAT_CHECK` | 判断是否重复 5 次 |
| `ST_FINISHED` | 完成 |
| `ST_ERROR` | 超时或异常 |

## 4. 程序结构

### 4.1 `User/main.c`

`main.c` 是核心业务流程，主要负责：
- 初始化电机1、电机2、按键、串口
- 处理按键触发
- 执行状态机
- 打印调试信息

### 4.2 电机1

电机1继续使用 A4988：
- `a4988_init()`
- `a4988_set_microstep()`
- `a4988_set_direction()`
- `a4988_run_rpm()`
- `a4988_stop_motor()`

电机1的动作还是保持原先的步进脉冲控制。

### 4.3 电机2

电机2已经改成闭环串口控制，设计目标是：
- 上电释放
- 可手动推动
- 读取实时位置
- 清零当前位置
- 回零

通讯使用 `USART2`：
- `PA2` = TX
- `PA3` = RX

## 5. 调试输出

串口调试输出仍保留，主要用于查看：
- 当前状态
- 电机1是否在运行
- 电机2当前位置
- 重复次数

示例：

```text
STATE=WAIT_M2 M1=0 M2=0 M2POS=123 CYCLE=2
```

## 6. 注意事项

1. `USART1` 和 `USART2` 不冲突，但必须分开接线。
2. 电机2闭环通讯需要确认波特率和协议帧格式与驱动器一致。
3. 如果光电传感器已经不用，可以从主程序中继续清理相关代码。
4. 当前工程里旧的“找两个传感器”逻辑已经不应该再作为主流程使用。
