# 任务规划

## 工作流程
1. **收集信息**：探索代码库并列出依赖项。
2. **起草计划**：
   - **目标**：任务的目标。
   - **方法**：高层策略。
   - **步骤**：顺序行动。
   - **验证**：如何证明它有效。
3. **澄清**：包括**至少 3 个澄清问题**，并提出答案。
4. **批准**：等待用户"前进"（空响应 = 批准）。
5. **实现**：仅在批准后执行。

## 原则
- **零影响**：规划期间不要修改文件。
- **迭代**：基于反馈完善。
- **简洁**：保持计划可读，通过 TODO 跟踪进度。
