# PY32F002B 指路方案验证板 README

本文档根据当前 `main.c` 的实际代码整理，描述工程功能、引脚分配、状态机、保护策略和联调注意事项，方便后续维护与验证。

## 1. 先说明当前代码的真实行为

当前 `main.c` 里有几处和旧说明不一致的地方，需要以代码为准：

- 挡位不是 `50% / 70% / 100%`，而是 `60% / 80% / 100%`。
- `APP_ADC_ENABLE = 0`，所以默认不做 ADC 采样，电流/电压保护逻辑在当前编译配置下不会真正生效。
- `APP_STALL_PROTECT_ENABLE = 0`，所以堵转/过流保护默认关闭。
- `APP_LOW_VBAT_PROTECT_ENABLE = 0`，所以低电压锁定默认关闭。
- `APP_BOOST_ENABLE` 已经做成可开关模块，关闭时会强制拉低 `PA5`。
- `APP_STOP_DELAY_MS = 5000` 相关 STOP 逻辑保留了框架，但 `APP_EnterStopMode()` 与 `APP_WakeupFromStop()` 目前是空实现。
- 代码里有部分中文注释编码显示异常，功能不受影响，但建议后续统一修复编码。

## 2. 程序总体功能

1. 上电后完成 `GPIO`、`TIM1 PWM`、串口调试初始化；`ADC` 仅在宏打开时初始化。
2. 通过 `SysTick` 产生 1ms 业务节拍，驱动周期任务：
   - 电流采样任务：20ms
   - 电池电压采样任务：100ms
3. 根据按钮、充电检测、电压、电流等信息构建状态机，统一控制：
   - 电机输出（`PA0`，`TIM1_CH1 PWM`）
   - 升压使能（`PA5`）
   - 三个指示灯（`PB0 / PB1 / PB3`）
4. 保留 `STOP` 低功耗流程的接口，但当前没有真正进入/退出 `STOP` 的实现。

## 4. 关键宏与默认配置

- `APP_UART_DEBUG_ENABLE = 1`
  - 开启串口调试输出，波特率 `115200`
- `APP_ADC_ENABLE = 0`
  - 默认关闭 ADC 采样
- `APP_STALL_PROTECT_ENABLE = 0`
  - 默认关闭堵转/过流保护
- `APP_LOW_VBAT_PROTECT_ENABLE = 0`
  - 默认关闭低电保护
- `APP_CHARGE_TASK_ENABLE = 1`
  - 开启充电状态检测
- `APP_STOP_ENABLE = 1`
  - 保留 STOP 逻辑入口，但实际函数为空
- `APP_BOOST_ENABLE = 1`
  - 升压模块开关，`1` 为启用自动升压控制，`0` 为关闭并强制拉低 `PA5`

## 5. 按键与挡位逻辑

### 5.1 按键行为

- 按住 `1.5s` 以上触发长按事件。
- 如果当前电机在运行：长按后进入停机延时流程。
- 如果当前电机未运行：长按后锁存启动请求。
- 松开按钮且不是长按触发时，会切换挡位。

### 5.2 挡位循环

挡位共 3 档，循环顺序为：

- 0 -> 1 -> 2 -> 0

对应的目标占空比为：

- 0 档：`60%`
- 1 档：`80%`
- 2 档：`100%`

### 5.3 电机启动方式

- 启动请求被锁存后，电机进入运行状态。
- 电机支持软启动：
  - 初始占空比：`100%`
  - 步进周期：`40ms`
  - 目标值由挡位决定

## 7. LED / 指示灯逻辑

### 7.1 三灯定义

- `PB0`：`LAMP1`
- `PB1`：`LAMP2`
- `PB3`：`LAMP3`

### 7.2 正常运行时

- 0 档：`PB0`
- 1 档：`PB0 + PB1`
- 2 档：`PB0 + PB1 + PB3`

### 7.3 充电中

- 未充满时：三灯做流水效果
  - `001 -> 010 -> 100`
- 充满后：两种状态交替
  - `010` 与 `101`

### 7.4 低电状态

| 条件 | 灯效 |
|---|---|
| 低电锁定进入时 | 三个挡位灯一起闪烁 `5` 次 |
| 闪烁完成后 | 三灯熄灭 |
| 低电释放后 | 恢复正常挡位显示 |

### 7.5 堵转/过流状态

| 挡位 | 灯效 |
|---|---|
| 1 档堵转 | `PB0` 闪烁，`PB1 / PB3` 熄灭 |
| 2 档堵转 | `PB0` 常亮，`PB1` 闪烁，`PB3` 熄灭 |
| 3 档堵转 | `PB0` 常亮，`PB1` 常亮，`PB3` 闪烁 |

### 7.6 故障/保护状态总结

- 低电锁定：三灯闪 `5` 次后熄灭
- 堵转/过流：当前挡位灯闪烁，其余更低挡位常亮、更高挡位熄灭
- 充电状态：流水灯或满电交替灯
- 正常状态：按挡位显示

## 8. 升压控制逻辑

`main.c` 中定义了升压阈值：

- 开启阈值：`3600mV`
- 关闭阈值：`3700mV`

当前已经做成模块化开关：

- `APP_BOOST_ENABLE = 1`：按阈值自动开关升压
- `APP_BOOST_ENABLE = 0`：关闭升压功能，并强制把 `PA5` 拉低

如果后续要临时禁用升压，只需要把宏改成 `0` 即可。

## 9. 电流、电压与保护逻辑

### 9.1 ADC 与换算

- 参考电压：`1500mV`
- 电流分流电阻：`5mΩ`
- 电池分压：`22k:10k`

### 9.2 电流采样

代码里预留了：

- ADC 原始采样
- 零点校准
- 有效值过滤
- 一阶低通滤波

但因为 `APP_ADC_ENABLE = 0`，这些逻辑默认不执行。

### 9.3 堵转 / 过流保护

预留了以下判定条件：

- 启动后 `500ms` 屏蔽
- 电流阈值：`6000mA`
- 连续命中 `20` 次后判定

但因为 `APP_STALL_PROTECT_ENABLE = 0`，默认不会触发。

### 9.4 低电保护

预留了：

- 进入阈值：`3200mV`
- 释放阈值：`3250mV`
- 连续低压确认：`2000ms`

但因为 `APP_LOW_VBAT_PROTECT_ENABLE = 0`，默认不会触发。

## 10. 低功耗与唤醒

代码里保留了 STOP 流程框架：

1. 按钮断开后，先等待 `5000ms`。
2. 若仍满足条件，调用 `APP_EnterStopMode()`。
3. 唤醒后调用 `APP_WakeupFromStop()`。

但当前这两个函数是空实现，所以实际上不会进入真正的低功耗控制流程。

## 11. 调试输出

开启串口调试后，会输出：

- 按钮状态
- 系统状态
- 电机状态
- 挡位
- 充电状态
- 低电锁定状态

当前已将周期状态打印调整为低频输出，避免串口拥塞。

## 12. 当前工程的用途总结

该工程是一个“电机 PWM + 按键换挡 + 充电检测 + 保护预留 + 低功耗框架”的验证样例，当前更接近调试框架而不是完整量产逻辑，特点如下：

- 串口调试已开启
- 按键长按/短按逻辑已实现
- 3 档挡位已实现，目标占空比为 `60% / 80% / 100%`
- 充电状态灯效已实现
- 低电灯效已改为三个挡位灯闪 `5` 次后熄灭
- 堵转灯效已改为“当前挡位灯闪烁，其余按档位状态显示”
- ADC、堵转保护、低电保护、升压控制的完整闭环仍处于预留或关闭状态
- STOP 低功耗接口保留，但未真正实现

## 13. 后续修改建议

如果后续准备把该工程变成可用版本，建议按下面顺序整理：

1. 统一注释编码，修复乱码字符。
2. 明确是否真的需要 `APP_ADC_ENABLE = 0` 的调试模式。
3. 若要启用保护，打开堵转和低电保护宏，并验证阈值。
4. 若要启用升压，只需把 `APP_BOOST_ENABLE` 改为 `1`。
5. 补全 `APP_EnterStopMode()` 和 `APP_WakeupFromStop()`。
6. 检查软启动参数，确认 `MOTOR_SOFT_START_*` 是否符合实际需求。
