此技能指导创建独特、生产级前端界面，避免通用"AI 垃圾"美学。实现真正工作的代码，对美学细节和创意选择给予卓越关注。

用户提供前端需求：要构建的组件、页面、应用程序或界面。他们可能包括关于目的、受众或技术约束的上下文。

## 设计思维

在编码之前，理解上下文并致力于大胆的美学方向：
- **目的**：此界面解决什么问题？谁使用它？
- **语气**：选择一个极端：残酷的极简主义、极致混乱、复古未来主义、有机/自然、奢华/精致、玩乐/玩具式、编辑/杂志式、野蛮主义/原始、艺术装饰/几何、柔和/粉色、工业/实用主义等。有很多风格可供选择。使用这些作为灵感，但设计一个真正符合美学方向的。
- **约束**：技术要求（框架、性能、可访问性）。
- **差异化**：是什么让这个界面难忘？人们会记住的一件事是什么？

**关键**：选择一个清晰的概念方向，并精确执行。大胆的极致主义和精致的极简主义都有效 - 关键是意图性，而不是强度。

然后实现工作代码（HTML/CSS/JS、React、Vue 等），该代码：
- 生产级且功能性
- 视觉上引人注目且难忘
- 与清晰的美学观点相一致
- 在每个细节上精心完善

## 前端美学指南

重点关注：
- **排版**：选择美丽、独特、有趣的字体。避免通用字体如 Arial 和 Inter；选择能提升前端美学的独特选择；意外的、有特色的字体选择。将独特的显示字体与精致的正文字体配对。
- **颜色和主题**：致力于一致的美学。使用 CSS 变量确保一致性。主要颜色与尖锐的强调色优于胆怯的、均匀分布的调色板。
- **运动**：使用动画进行效果和微交互。优先使用 HTML 的纯 CSS 解决方案。React 时使用 Motion 库。专注于高影响时刻：一个精心编排的页面加载与交错揭示（animation-delay）比分散的微交互创造更多愉悦。使用滚动触发和悬停状态来惊喜。
- **空间构图**：意外的布局。不对称。重叠。对角流动。打破网格的元素。慷慨的负空间或受控密度。
- **背景和视觉细节**：创造氛围和深度，而不是默认使用纯色。添加与整体美学匹配的上下文效果和纹理。应用创意形式如渐变网格、噪声纹理、几何图案、分层透明度、戏剧性阴影、装饰边框、自定义光标和颗粒覆盖。

永远不要使用通用 AI 生成的美学，如过度使用的字体家族（Inter、Roboto、Arial、系统字体）、陈词滥调的配色方案（特别是白色背景上的紫色渐变）、可预测的布局和组件模式，以及缺乏上下文特定特征的饼干切割设计。

创造性地解释并做出意外的选择，让人感觉真正为上下文设计。没有任何设计应该是相同的。在浅色和深色主题、不同字体、不同美学之间变化。永远不要在世代中收敛到常见选择（例如 Space Grotesk）。

**重要**：将实现复杂性与美学愿景匹配。极致主义设计需要精心设计的代码，具有广泛的动画和效果。极简主义或精致设计需要克制、精确和对间距、排版和微妙细节的仔细关注。优雅来自良好执行愿景。
